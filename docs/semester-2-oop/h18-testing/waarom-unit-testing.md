# Waarom Unit Testing?

![](/img/img-3249.png)


Door gebruik te maken van Unit Testing:

* Zullen fouten snel worden gevonden\
  We testen kleine stukken code afzonderlijk vooraleer ze samen worden getest\
   
* Krijgen we betere code\
  Door het maken van unit tests ga je extra nadenken over de code en alle randvoorwaarden en negatieve paden.<br/>
* Bekomen we extra documentatie\
  De unit testen geven aan hoe een component moeten worden gebruikt (net zoals je zou gaan zoeken op het internet naar code voorbeelden over een bepaalde component.)<br/>
* Hebben we een extra zekerheid wanneer bestaande code moet worden aangepast\
  \- bij het oplossen van een bug\
  \- bij het toevoegen van nieuwe functionaliteit\
  \- bij het refactoring van een stuk bestaande code of een component\
